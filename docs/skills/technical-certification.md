# Technical Certification Skill

Use this skill for cloud, security, networking, developer, or operations
certification prep.

Pair it with `study-card-maintainer.md`. For Markdown notes, command snippets,
or exported documentation summaries, also use `markdown-to-deck.md`.

## Output

- Cards for acronyms, service boundaries, command meanings, and failure modes.
- Scenario cards for troubleshooting and architecture choices.
- Text entries for short explainers that are useful for audio review.
- Notes that separate facts, high-probability inferences, and unknowns.

## Agent Checklist

1. Identify the vendor, tool, exam, version, or date context when available.
2. Extract acronyms, service boundaries, limits, and failure modes.
3. Create scenario cards only when the source supports the recommended action.
4. Keep commands and configuration examples exact when they come from source.
5. Separate source facts from inferred explanations in `detail`.
6. Mark version-sensitive or uncertain claims as `needs-review`.

## Rules

- Prefer official or user-provided source material.
- Avoid vendor-specific claims unless the source says so.
- Include version or date context when a rule may change.
- Keep generated examples clearly separate from source quotations.

## Useful Tags

- `acronym`
- `service-boundary`
- `troubleshooting`
- `architecture`
- `command`
- `needs-review`
